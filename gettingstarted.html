      Where to start?

      The best place to start is to brainstorm what type of connectivity is required for the current project. Then, after a plan is formulated, select a programming language that would work best with the plan. Once a language is selected, look at the models available for the selected language to see which would fit the communication style. Each model provides an example of a communication style that works with a microcontroller. The code is modularly constructedso it can be easily incorporated into any project. Below is the list of materials needed to use any of the arduivis patches. This list is periodically updated.

      Requirements

      • Microcontroller: Arduino, Adafruit Trinket, RBBB (Reall Bare Bones Board)
      • Programming Language: MaxMSP, PD-Extended
      • Download the programming environment: Arduino IDE, Sublime Text
      • Download the repository: arduivis

      Model Types

      Examples of these model types can be found in the selected programming language

      • output: This is the flow of data from a microcontroller to a programming language
      • input: This is the flow of data from a programming language to a microcontroller
      • feedback This is the bi-directional flow of data between a programming language and a microcontroller
      • multipleOutputs: Multiple streams of data going from a microcontroller to a programming language
      • multipleInputs: Multiple streams of data going from a programming language to a microcontroller
      • multipleFeedbacks: Multiple streams of bi-directional between a programming language and a microcontroller
